# tiffix

TIFF画像列に対する双方向スキャン歪み補正を行うGUIツール。

## Install

このプロジェクトは Python パッケージマネージャー `uv` で管理されています。  
依存関係のインストールや実行には `uv` を使用してください。

まず、各自の環境に `uv` をインストールしてください。  
インストール方法は[公式ドキュメント](https://docs.astral.sh/uv/)を参照してください。  

もしも`git`が各自の環境になければ、あらかじめインストールしてください。  
インストール方法は[公式ドキュメント](https://git-scm.com/install/linux)を参照してください。

### `uv`と`git`のインストール後

以下のコマンドを実行してgithubからソースコードをダウンロードしてください。  
```
git clone https://github.com/SomeKindOfDuck/tiffix tiffix
cd tiffix
```

ターミナル上で`uv sync`を実行して依存関係をインストールします。  
問題なくインストールできたら、`uv run tiffix`を実行して、GUIが立ち上がります。  

上記のコマンドでもtiffixを使用することはできますが、プロジェクト直下で`uv tool install .`を実行することで、ワーキングディレクトリに関わらず`tiffix`でGUIを立ち上げることができるようになります。  

## 使い方

GUIを起動したら、まず右上の Select image directory をクリックします。 
ディレクトリ選択画面が表示されるので、ズレ補正を行いたいTIF画像が保存されているディレクトリを選択してください。  

選択したディレクトリ内にTIFファイルが存在しない場合はエラーが表示されます。その場合は再度 Select image directory をクリックして、正しいディレクトリを選択し直してください。  
正常に選択できると、ボタンの下に現在のディレクトリ名が表示されます。  

ディレクトリを選択すると、内部のファイルがソートされ、最初の画像が画面に表示されます。  
画面には同じ画像が左右に表示され、左が補正前、右が補正後の画像です。  

### パラメータ設定

ズレ補正に使用する主なパラメータは以下の3つです。  

+ Start frame for averaging: 表示に使用する開始フレーム番号を指定します。
+ Frames to average: 指定した開始フレームから何枚の画像を平均化するかを指定します。
+ Horizontal shift: 奇数行・偶数行のずれを補正するためのシフト量（ピクセル）を指定します。

#### 平均化パラメータの動作

例えば、Frames to average = 1 の場合、  
Start frame for averaging で指定した1枚の画像がそのまま表示されます。

一方、Frames to average ≥ 2 の場合は、  
開始フレームから指定枚数分の画像を読み込み、それらを平均化した1枚の画像が表示されます。

パラメータを変更した後は、Reload image ボタンを押すことで表示が更新されます。  
また、Auto reload を有効にすると、パラメータ変更に応じて自動的に画像が更新されます。  

ただし、平均化に使用する枚数が多い場合は処理に時間がかかるため、動作が遅くなることがあります。  

#### ズレ補正の調整

平均化パラメータを調整して、細胞などの構造が見やすい画像を表示したら、Horizontal shift を調整してズレ補正を行います。

このパラメータを変更すると、画像の奇数行と偶数行のいずれかが指定したピクセル数だけ水平方向にシフトされます。  
値は正負の両方を取るため、ズレが悪化した場合は反対方向に調整してください。  

ズレの確認が難しい場合は、マウス操作で画像の拡大・移動が可能です。  
ホイールでズーム、ドラッグで移動ができるので、細部を確認しながら調整してください。  

奇数行と偶数行のズレが解消されれば、補正は完了です。  

#### 画像の保存

画面下部の Save image ボタンを押すと、現在のパラメータがすべての画像に適用され、ズレ補正が実行されます。  

補正後の画像は、選択したディレクトリ内に新しく作成される corrected フォルダに保存されます。
